PROJECT OVERVIEW
----------------
This project is a retrieval middleware proof-of-concept built with FastAPI. It sits between a user query and a (mock) vector database, adds a semantic cache layer, and re-ranks candidate documents with a cross-encoder. The core flow for POST /query is:
1) Embed the query into a vector.
2) Try a semantic cache lookup.
3) On a cache hit: return cached results immediately.
4) On a cache miss: query the vector DB for similar documents, re-rank them with a cross-encoder, cache the result, and return it along with detailed latency breakdowns.


FILE-BY-FILE EXPLANATION
------------------------

config.py
---------
- CACHE_SIMILARITY_THRESHOLD = 0.9
  - Minimum cosine similarity between the current query embedding and a cached query embedding required to treat it as a "semantic cache hit". 0.9 is high, meaning only very similar queries will share results.
- CACHE_MAX_SIZE = 100
  - Maximum number of entries the in-memory semantic cache keeps. When full, it evicts the oldest entry (FIFO).
- VECTOR_DB_TOP_K = 50
  - Number of top documents to retrieve from the mock vector database before re-ranking.
- RERANKER_TOP_N = 5
  - Default number of final results the reranker returns if the request does not specify top_n.
- EMBEDDING_MODEL = "all-MiniLM-L6-v2"
  - Sentence-transformers model used to convert query text into a 384-dimensional embedding. This embedding is used both for cache similarity checks and vector DB search.
- RERANKER_MODEL = "cross-encoder/ms-marco-MiniLM-L-6-v2"
  - Cross-encoder model used to re-score (query, document_text) pairs for more accurate ranking.
- EMBEDDING_DIM = 384
  - Expected dimensionality of all embeddings; validates that generated embeddings have the correct size.
- MOCK_DB_LATENCY_MIN_MS = 80 / MOCK_DB_LATENCY_MAX_MS = 150
  - Lower/upper bounds (milliseconds) for simulated network latency when calling the mock vector DB.


vector_db_client.py
-------------------
Purpose: Provide a mock vector database client that behaves like a hosted vector DB (Pinecone/Milvus/etc.), but purely in memory.

- Imports numpy, asyncio, random, and project config constants.
- class VectorDBClient:
  - __init__(self, num_documents: int = 256):
    - Uses a numpy random generator with a fixed seed (42) for reproducible embeddings.
    - Creates raw_embeddings: an array of shape (num_documents, EMBEDDING_DIM) with random values.
    - Normalizes each embedding vector to unit length (L2 norm), storing the result in self._embeddings. This means cosine similarity can be computed as a dot product.
    - Builds a list of document metadata (self._documents), each with:
      - id: "doc-i"
      - text: "Document i about topic (i % 10)."
    - There is no external DB; all data is generated at startup and kept in memory.
  - async def query(self, embedding: np.ndarray, top_k: int = VECTOR_DB_TOP_K):
    - Simulates network latency:
      - Picks a random latency between MOCK_DB_LATENCY_MIN_MS and MOCK_DB_LATENCY_MAX_MS.
      - Awaits asyncio.sleep(latency_ms / 1000.0).
    - Validates the input embedding dimension against EMBEDDING_DIM.
    - Normalizes the query embedding to unit length.
      - If the norm is zero, raises an error (zero-length vector cannot be normalized).
    - Computes cosine similarity scores by dot product: scores = self._embeddings @ query_vec.
    - Finds indices of the top_k highest scores in descending order.
    - Builds and returns a list of result dicts: { "id", "text", "score" }.


semantic_cache.py
-----------------
Purpose: Provide an in-memory semantic cache that stores query embeddings and their results, and can return cached results when a new query is similar enough to a previous one.

- @dataclass CacheEntry:
  - embedding: numpy array representing the cached query embedding.
  - results: list of document dicts (as returned from the pipeline).

- class SemanticCache:
  - __init__(max_size=CACHE_MAX_SIZE, similarity_threshold=CACHE_SIMILARITY_THRESHOLD):
    - Stores:
      - self._entries: a deque of CacheEntry objects (acts as the FIFO cache).
      - self._hit_count / self._miss_count: counters for cache stats.
    - max_size: maximum number of cache entries to keep.
    - similarity_threshold: minimum cosine similarity for a cache hit.
  - _cosine_similarity(a, b):
    - Static helper that:
      - Ensures both vectors have the expected dimension.
      - Computes cosine similarity = dot(a, b) / (||a|| * ||b||).
      - Handles zero-norm vectors safely by returning 0.0.
  - get(self, query_embedding):
    - Iterates over all cache entries.
    - Computes similarity between the query_embedding and each entry.embedding using _cosine_similarity.
    - Tracks the entry with the best (highest) similarity and its results.
    - If the best similarity >= threshold:
      - Increments hit_count.
      - Returns (True, best_results, best_similarity).
    - Otherwise:
      - Increments miss_count.
      - Returns (False, None, best_similarity).
  - set(self, query_embedding, results):
    - If the cache is at max_size, evicts the oldest entry (popleft).
    - Appends a new CacheEntry with a copy of the embedding and the results list.
  - stats(self):
    - Returns a dict with:
      - hit_count, miss_count,
      - hit_rate (% of queries that were hits),
      - current_size (number of entries currently stored).
  - clear(self):
    - Empties the cache and resets hit/miss counters to zero.


reranker.py
-----------
Purpose: Re-rank the top-k vector DB results using a Cross-Encoder model for better relevance scoring.

- Imports CrossEncoder from sentence-transformers and config constants.

- class Reranker:
  - __init__(model_name=RERANKER_MODEL):
    - Loads the cross-encoder model at initialization: CrossEncoder(model_name, max_length=512).
  - rerank(self, query, documents, top_n=RERANKER_TOP_N):
    - If documents is empty, returns ([], 0.0) immediately.
    - Builds pairs = [(query, doc["text"]) for doc in documents].
    - Records start time, calls self._model.predict(pairs) to get scores for each pair, and computes elapsed time in ms.
    - Constructs a new list of docs with updated "score" from the cross-encoder.
    - Sorts the docs by descending score.
    - Slices to the requested top_n (bounded by the number of available docs).
    - Returns (top_docs, elapsed_ms).


models.py
---------
Purpose: Pydantic models that define the shapes of requests and responses for FastAPI.

- QueryRequest:
  - query: string input from the user.
  - top_n: optional integer (default 5, must be > 0). Controls how many final re-ranked docs to return.

- Document:
  - id: document ID.
  - text: document content/summary.
  - score: float score (for the current stage, e.g., reranker score).

- LatencyBreakdown:
  - embedding_ms: time spent embedding the query.
  - cache_lookup_ms: time spent in semantic cache lookup.
  - vector_db_ms: time spent querying the vector DB (0 if cache hit).
  - reranking_ms: time spent re-ranking (0 if cache hit).
  - total_ms: total end-to-end time for the /query request.

- QueryResponse:
  - results: list[Document] of size top_n (or fewer if fewer docs available).
  - cache_hit: bool indicating if the response was served from the semantic cache.
  - cache_similarity_score: highest similarity score from cache lookup (even on a miss).
  - latency_breakdown: LatencyBreakdown object summarizing timings.

- StatsResponse:
  - hit_count, miss_count, hit_rate, current_size: directly mirror SemanticCache stats.
  - from_cache_stats(cls, stats): helper to map a raw stats dict into a strongly-typed StatsResponse.


main.py
-------
Purpose: FastAPI application entrypoint, wiring together embedding, cache, vector DB, and reranker into a single /query endpoint plus cache stats endpoints.

- Imports:
  - FastAPI for API routing.
  - SentenceTransformer for embedding.
  - Numpy for vector math.
  - Config/constants, models, semantic_cache, vector_db_client, reranker.

- Global objects:
  - app = FastAPI(title="Retrieval Middleware POC")
    - The FastAPI application object.
  - _embedding_model = None / _reranker = None
    - Globals used for lazy-loading heavy models so the server binds quickly. Actual model instances are created on first use.
  - vector_client = VectorDBClient()
    - Global mock vector DB client used by all requests.
  - cache = SemanticCache()
    - Global semantic cache shared across requests.

- Lazy-load helpers:
  - _get_embedding_model():
    - If _embedding_model is None, loads SentenceTransformer(EMBEDDING_MODEL) and stores it.
    - Returns the embedding model instance.
  - _get_reranker():
    - If _reranker is None, creates Reranker() and stores it.
    - Returns the reranker instance.

- _embed_query(query: str) -> np.ndarray:
  - Calls _get_embedding_model().encode(query, convert_to_numpy=True) to get a numpy embedding.
  - Validates that the embedding length == EMBEDDING_DIM; raises if not.
  - Returns the embedding vector.

- @app.post("/query", response_model=QueryResponse):
  - query_endpoint(request: QueryRequest) -> QueryResponse
  - Pipeline:
    1) Record total_start time.
    2) Embedding stage:
       - Time the call to _embed_query.
       - Record embedding_ms.
    3) Cache lookup:
       - Call cache.get(embedding).
       - Get hit (bool), cached_results (or None), and similarity_score.
       - Record cache_lookup_ms.
    4) Initialize vector_db_ms and reranking_ms = 0.0, results = [].
    5) If hit and cached_results is not None:
       - Build Document Pydantic objects from cached dicts, up to request.top_n.
       - Skip vector DB and reranker entirely.
    6) Else (cache miss):
       - Vector DB:
         - Time vector_client.query(embedding, top_k=VECTOR_DB_TOP_K).
         - Record vector_db_ms.
       - Re-ranker:
         - Time _get_reranker().rerank(request.query, candidates, top_n=request.top_n or RERANKER_TOP_N).
         - Build Document objects from the returned list.
         - Record reranking_ms.
       - Cache storage:
         - cache.set(embedding, [doc.model_dump() for doc in results]).
           (Uses model_dump() to turn Document Pydantic models into plain dicts.)
    7) Compute total_ms for the entire request.
    8) Build a LatencyBreakdown object from all timing values.
    9) Return a QueryResponse with:
       - results: final Document list.
       - cache_hit: whether the semantic cache was used.
       - cache_similarity_score: similarity from cache.get.
       - latency_breakdown: detailed timings.

- @app.get("/stats", response_model=StatsResponse):
  - stats_endpoint():
    - Calls cache.stats() and wraps it into a StatsResponse via from_cache_stats.
    - Returns global cache metrics: hit/miss counts, hit rate, current cache size.

- @app.delete("/cache"):
  - clear_cache_endpoint():
    - Calls cache.clear().
    - Returns a simple {"status": "ok"} JSON response.


tests/test_cache.py
-------------------
Purpose: Unit tests for the semantic cache behavior and for the /stats endpoint.

- Imports:
  - numpy for building embeddings.
  - pytest for assertions like pytest.approx.
  - FastAPI TestClient for calling the API synchronously.
  - EMBEDDING_DIM, app, cache, SemanticCache.

- client = TestClient(app):
  - Creates a synchronous test client that can send HTTP requests to the FastAPI app without starting a real server.

- test_identical_query_cache_hit_similarity_one():
  - Creates a fresh SemanticCache.
  - Builds an embedding of all ones.
  - Stores a simple results list.
  - Calls semantic_cache.get(embedding).
  - Asserts:
    - hit is True.
    - Returned results equal what was stored.
    - Similarity is approximately 1.0 (within 1e-6), meaning identical vectors compare to 1.0 cosine similarity.

- test_semantically_different_query_cache_miss():
  - Creates a SemanticCache and stores an all-ones embedding.
  - Creates a different embedding of all zeros.
  - Calls get(different_embedding).
  - Asserts:
    - hit is False.
    - cached_results is None.
    - similarity == 0.0 (orthogonal/zero vector).

- test_cache_eviction_fifo():
  - Creates a SemanticCache with max_size = 3.
  - Inserts 4 different embeddings, each with its own doc entry.
  - Because capacity is 3, the oldest entry is evicted.
  - Asserts that current_size == max_size (3), confirming FIFO eviction.

- test_stats_endpoint_reflects_hit_miss_counts():
  - Clears the global cache.
  - Sends one POST /query (cold query) via client.post, which should be a cache miss.
  - Sends another POST /query with the same query, producing a hit.
  - Calls GET /stats.
  - Asserts:
    - Status code 200.
    - hit_count >= 1, miss_count >= 1.
    - current_size >= 1 (at least one cached entry exists).

WHAT /docs IS SUPPOSED TO OUTPUT
--------------------------------
The /docs path is automatically provided by FastAPI using Swagger UI. Technically:
- FastAPI inspects all defined routes (e.g., POST /query, GET /stats, DELETE /cache) and the Pydantic models used as request/response bodies.
- It generates an OpenAPI specification (JSON describing endpoints, parameters, schemas, and responses).
- /docs serves a single-page web application (Swagger UI) that:
  - Reads the OpenAPI spec from /openapi.json.
  - Renders interactive documentation listing each endpoint.
  - Shows the request/response JSON schema for each route (based on QueryRequest, QueryResponse, StatsResponse, etc.).
  - Lets you try endpoints directly in the browser by filling forms and sending live requests against the running server.

So when you visit http://127.0.0.1:8000/docs you should see an interactive UI listing:
- POST /query
- GET /stats
- DELETE /cache
with models and example payloads derived from the code.


WHAT THE TESTS ARE DOING AND HOW WE KNOW THEY PASSED
----------------------------------------------------
Tests are split into three concerns:
1) Cache behavior (tests/test_cache.py)
2) Reranker behavior (tests/test_reranker.py)
3) End-to-end middleware flow (tests/test_middleware.py)

They collectively verify:
- Cosine similarity math and cache hit/miss semantics.
- FIFO eviction when the cache is full.
- The /stats endpoint exposes accurate hit/miss counters.
- Reranker returns correct top_n counts, sorted by descending score, and gracefully handles "fewer docs than requested".
- The full middleware pipeline:
  - Cold query executes embedding, vector DB, reranker, and returns correct latency fields.
  - Warm repeated query uses semantic cache (no vector DB or reranking work).
  - Semantically similar queries hit the cache based on similarity threshold.
  - Clearing cache resets behavior so subsequent queries are cold misses again.

We know they all passed because pytest was run from the project root with:
- PYTHONPATH=. pytest retrieval_middleware/tests -v
and the output showed:
- 11 tests collected.
- All 11 tests with status PASSED.
- A final summary line similar to:
  "11 passed, 30 warnings in XX.XXs"
No FAILED or ERROR lines remained. The earlier failures were fixed by:
- Importing pytest in test_cache.py.
- Slightly lowering CACHE_SIMILARITY_THRESHOLD so the paraphrased query test reliably hits the cache.
